TLDR: My team and I developed a lane detection algorithm using OpenCV & Python. A green bounding box was used to represent the area we are confident we can drive in.

Thresholding: saturation, HSL conversion, bilateral blur, color mask, edge detection, bitwise AND/OR for mask blur, ROI selection.
Polyfit: perspective warp, main contour selection, polyfit, MSE + confidence calculation, acceptance/rejection decision, filtering of polyline coefficients



Full Story: In my third year at Northeastern University, I enrolled in a graduate level course taught by Professor Hanumaunt Singh. Our final project for the class was to develop a lane detection algorithm, working off of Medhi Squalli's code found on Medium. My team dove into this project thinking it would be easy since we had some code to work off of. We were wrong.



As you can see from the images, Medhi used very perfect images and videos. These images are from perfect driving conditions: sunny, no reflection, highway, straight lanes. We set out to collect more data and recorded ourselves driving in many different conditions such as: rain, snow, curved lanes, night time, shadows, switching lanes, etc. We quickly realized that Medhi's code broke for almost every other case, which meant that Medhi's code was almost useless to us. We did however, learn more about thresholding and perspective warping from his write up, which served as a good starting point for my group.

The first step to detecting lanes on an image is to segment the image so that we only focus on the region that intrests us. The first step in our thresholding was saturating our image to make the white/yellow lane lines more noticable in our edge detector. Next we converted our image to HSL color space (unlike Medhi who used HSV).
